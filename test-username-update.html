Ejemplos de Uso del Servicio

1. Actualización Básica con Preview

// Frontend - JavaScript const result = await usernameUpdateService.previewUsernameUpdate( '1411cebf-62c0-46bc-8e73-e5b121604aa2', 'nuevo-username' ); console.log('Referencias encontradas:', result.data.contentReferences.total); console.log('Puede proceder:', result.data.canProceed);

2. Actualización Completa con Todas las Opciones

// Frontend - JavaScript const result = await usernameUpdateService.updateUsername( '1411cebf-62c0-46bc-8e73-e5b121604aa2', 'nuevo-username', { timeout: 30000, retries: 3, skipCacheInvalidation: false, skipContentUpdate: false, skipRedirects: false } ); if (result.success) { console.log('Username actualizado exitosamente'); console.log('Tiempo de ejecución:', result.data.executionTimeMs + 'ms'); }

3. Validación de Username

// Frontend - JavaScript const validation = usernameUpdateService.validateUsername('nuevo-username'); if (!validation.isValid) { console.log('Errores:', validation.errors); } else { console.log('Username válido'); if (validation.warnings.length > 0) { console.log('Advertencias:', validation.warnings); } }

4. Obtener Estadísticas

// Frontend - JavaScript const stats = await usernameUpdateService.getUpdateStatistics( '1411cebf-62c0-46bc-8e73-e5b121604aa2', { useCache: true } ); console.log('Total de actualizaciones:', stats.data.totalUpdates); console.log('Redirecciones activas:', stats.data.redirectsActive);

5. Suscripción a Eventos Globales

Documentación del Sistema

Arquitectura del Sistema

El sistema de actualización de nombres de usuario está compuesto por:

  • UsernameUpdateService (Backend): Servicio principal que coordina todas las operaciones
  • ContentUpdateService: Actualiza referencias en comentarios, publicaciones, mensajes privados
  • CacheInvalidationService: Gestiona la invalidación de caché distribuida
  • ValidationService: Valida reglas de negocio y formato

Flujo de Actualización

  1. Validación: Se valida el formato y disponibilidad del nuevo username
  2. Preview (opcional): Se obtiene información de referencias sin modificar datos
  3. Transacción: Se inicia una transacción de base de datos
  4. Actualización de contenido: Se actualizan todas las referencias en el sistema
  5. Actualización de usuario: Se cambia el username en la tabla de usuarios
  6. Redirecciones: Se crean redirecciones 301 para URLs antiguas
  7. Invalidación de caché: Se limpia caché relacionada con el usuario
  8. Commit: Se confirman todos los cambios

Características de Seguridad

  • Transacciones atómicas: Todas las operaciones se realizan en una sola transacción
  • Rollback automático: Si ocurre un error, se revierten todos los cambios
  • Validación estricta: Se valida el formato y disponibilidad del username
  • CSRF Protection: Todas las peticiones requieren token CSRF válido
  • Timeouts configurables: Se puede configurar timeouts por operación
  • Reintentos automáticos: Sistema de reintentos para fallos temporales

Tipos de Referencias que se Actualizan

  • Comentarios: Referencias @username en comentarios
  • Publicaciones: Menciones en publicaciones y páginas
  • Mensajes privados: Referencias en mensajes directos
  • Feed: Menciones en el feed de actividad
  • URLs: Creación de redirecciones 301 automáticas
